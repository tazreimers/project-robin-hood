# 2. Add bootstrap scripts

Add a root-level scripts directory:

scripts/
  bootstrap.sh
  dev.sh
  test.sh
  lint.sh
  format.sh
  reset_local.sh
  seed_local.sh
  check_env.sh

Each script should be POSIX-friendly bash.

bootstrap.sh should:
- check Docker is installed
- check Docker Compose is available
- check Node is installed if frontend needs local install
- check Python is installed if backend needs local tooling
- create .env from .env.example if missing
- print clear next steps
- avoid overwriting existing .env

dev.sh should:
- start the full local stack with Docker Compose
- print frontend URL
- print backend URL
- print API docs URL

test.sh should:
- run backend tests
- run frontend tests if configured
- print clear pass/fail status

lint.sh should:
- run backend linting if configured
- run frontend linting if configured

format.sh should:
- run backend formatting if configured
- run frontend formatting if configured

reset_local.sh should:
- warn the user
- reset local Docker volumes/database
- rerun migrations
- optionally seed local data

seed_local.sh should:
- add useful local demo data if seed support exists
- create a small set of fake sports, events, bookmakers, odds, and arbitrage opportunities
- make the UI usable without a live API key

check_env.sh should:
- validate required environment variables
- print missing variables
- explain which are optional

Update file permissions so scripts are executable.
